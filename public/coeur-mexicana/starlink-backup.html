01 — Antecedentes

Necesidad del Cliente

Solución propuesta

Sistema de backup con 2 terminales Starlink conectadas a un router empresarial de alta disponibilidad con failover automático. Túnel VPN seguro hacia servidor en la nube para proporcionar IP fija única que IT Palo Alto pueda autorizar permanentemente.

🔒

Requerimiento IT corporativo

IT Palo Alto requiere una IP fija pública única para whitelist en sus firewalls. No importa qué Starlink esté activo, el tráfico siempre debe salir por la misma IP para cumplir con políticas de seguridad corporativas.

📊

Gestión de datos

Starlink 1 tiene 1 GB de datos mensuales y Starlink 2 tiene 4 GB. Total: 5 GB para contingencias. El sistema de gestión de red existente manejará QoS y control de tráfico para optimizar el uso de datos durante las contingencias.

03 — Operación

Flujo de Datos

Operación normal — Enlace RF/Microondas activo

Todo el tráfico de Palmarejo sale por el enlace principal RF/Microondas que está conectado directamente a la red LAN. Los Starlinks permanecen en standby consumiendo datos mínimos. El sistema de backup está listo pero inactivo.

Detección de falla — Se detecta caída del enlace principal

Cuando el enlace RF/Microondas presenta fallas, la red LAN activa la conexión de respaldo a través del router empresarial TRN. El sistema comienza el proceso de failover automático.

Activación de backup — Túnel VPN se establece

El router TRN activa Starlink 1 y establece el túnel VPN seguro hacia el servidor cloud. El tráfico ahora fluye: LAN → Router TRN → Starlink → Túnel VPN → Servidor Cloud (IP fija). IT Palo Alto ve la IP fija autorizada.

Failover secundario — Si Starlink 1 agota datos

Si Starlink 1 (1 GB) se agota o presenta problemas, el router TRN automáticamente conmuta a Starlink 2 (4 GB). El túnel VPN se restablece transparentemente y la IP sigue siendo la misma.

Recuperación — Enlace principal restaurado

Cuando el enlace RF/Microondas se recupera, el tráfico regresa automáticamente al enlace principal. Los Starlinks vuelven a standby y el router TRN queda listo para la próxima contingencia.

📡 Terminales Starlink

Ambas terminales Starlink se conectan a los puertos WAN del router empresarial TRN (puertos 1 y 2). Permanecen en modo standby hasta que el enlace principal RF/Microondas presenta fallas, activándose automáticamente como respaldo.

🔀 Router Empresarial TRN

El router de alta disponibilidad gestiona exclusivamente el sistema de backup Starlink. Cuenta con 4 puertos WAN y 4 puertos LAN, preparado para futuras expansiones. Maneja el failover automático entre Starlinks y establece el túnel VPN seguro.

📻 Enlace RF/Microondas

El enlace principal RF/Microondas está conectado directamente a la red LAN de Palmarejo, integrado con la infraestructura existente. Este enlace opera de forma independiente al sistema de backup TRN.

🔒 Túnel VPN Seguro

El túnel VPN encripta todo el tráfico entre el router TRN y el servidor cloud utilizando criptografía de grado militar (AES-256). Se restablece automáticamente si cambia la conexión WAN activa entre Starlinks.
